## Summary

### Key Lessons
- Refined module design: trains enter from the top and exit downward, matching right-hand-drive conventions for cleaner flow
- The module produces 723 plates/second (9 smelting rows x 16 furnaces x 5.025 plates/second)
- A dedicated fuel station must be included within the module -- cannot rely on manual fueling
- Rearranging the module entry/exit to avoid unnecessary crossings dramatically simplifies the rail layout
- Taking existing smelting offline to build new modules causes temporary science production crashes

### Design Principles
- Train direction: inbound from top, outbound downward = natural right-hand-drive flow with no crossing conflicts
- Fuel station placement: integrate within the module rather than bolting on externally
- When redesigning, prioritize eliminating rail crossings -- each crossing is a potential deadlock point
- Symmetrical module layout enables easier duplication and mental model of train flow
- Always ensure chain signal blocks are long enough to hold a full train; 2-segment trains need at least 8 tiles between signals

### Ratios & Numbers
- 9 smelting stops per module, 17 trains total (10 input + 4 output + 4 buffer - 1)
- 723 copper/iron plates per second per module = approximately twice the output of old-style smelting
- 5.025 plates/second per beaconed furnace at current productivity level
- Low-density structures shortage cascades to rockets -> white science -> total science shutdown
- Module footprint: 28 tiles wide, tileable vertically with snap-to-grid

### Mistakes to Avoid
- Do not forget to include a fuel delivery solution in the module design -- trains will run out and the whole loop stops
- Taking old smelting offline before new modules are fully operational causes resource crashes
- Rail crossings within the module that are "rarely used" still need proper chain signal protection
- Do not mix old-style and new-style train station naming -- it creates routing confusion in the train network
