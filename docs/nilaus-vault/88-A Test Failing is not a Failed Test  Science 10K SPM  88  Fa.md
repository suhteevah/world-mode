## Summary

### Key Lessons
- A failed test is not a failed project; testing things that do not work is a valuable part of megabase building
- Transport line gaps (partially filled belts) are a significant UPS cost; full belts are treated as single calculation entities while gaps create multiple entities
- The debug menu (F5) reveals that "entity update" and "transport lines" are the two biggest UPS consumers
- Reducing splitter count and eliminating half-empty belts in station designs can meaningfully improve UPS
- Building designs should minimize the number of belt segments that are only partially full

### Design Principles
- Use the "show transport line gaps" debug option to identify UPS-expensive belt configurations
- Full belts = efficient (one calculation entity); partial belts = expensive (multiple calculation entities per gap)
- Redesign unloading stations to minimize splitters and keep belts saturated; direct insertion where possible
- Iteration and reverting failed builds is a normal part of megabase development; do not treat it as failure
- Recording video costs some UPS; benchmark numbers while recording are slightly lower than actual capability

### Ratios & Numbers
- UPS while recording: ~42 (slightly lower than non-recording baseline)
- Purple science: running at 10,000/min (fully scaled)
- Blue circuits: 96,000 sitting idle in buffers (64K at two stations + full trains waiting) confirming oversupply
- Red, blue, green, purple science: all running at 10,000 SPM
- No dashboard warnings = fully saturated supply chains for all scaled sciences

### Mistakes to Avoid
- Do not use belt balancers with many splitters at every station; each splitter creates transport line gaps that cost UPS
- Do not view reverting a build as failure; it is a successful test that revealed what does not work
- Do not ignore the transport line debug view; it reveals hidden UPS costs invisible in normal gameplay
- Do not forget that recording overhead affects UPS measurements; always note whether benchmarks are during recording
